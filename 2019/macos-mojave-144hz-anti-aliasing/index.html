MacOS Mojave 144Hz + Anti-Aliasing

2019-01-11
macOS

Sommaire

    • The problem
    • The Solution : Displayport
    • WTF, Anti-Aliasing en vacances ?

Depuis quelques semaines j’utilise un Macbook Pro comme ordinateur principal, relié a deux écrans via des adaptateurs classiques USB-C vers HDMI.

  • Le premier est un écran LG (IPS) 1080p @ 60Hz.
  • Le second est un BenQ (TN) 1080p @ 144Hz.

Dans l’optique de jouer dans de bonnes conditions avec Shadow-PC, j’ai voulu passer mon écran BenQ de 60Hz à 144Hz… Et là c’est le drame, ça tourne mal, la suite va vous étonner !

The problem

Impossible d’obtenir une fréquence de rafraîchissement > 60Hz

Bon, ça commence mal car dans les options “Moniteurs” je n’ai pas la possibilité d’augmenter la fréquence de rafraichissement qui semble déjà au maximum : 60Hz.

Pourtant, c’est pas faute d’avoir testé moult cables et adaptateurs :

  • Les fameux multi-adaptateurs USB-C hors de prix… (ref. Amazon)
  • Un câble USB-C <> HDMI 2.1 4K@60Hz et 1080p@144Hz et compatible Macbook Pro 2018 et Thunderbolt 3 (ref. Amazon)

The Solution : Displayport

J’ai finalement réussi avec un câble vers Displayport (ref. Amazon)

It works ! 🤩

WTF, Anti-Aliasing en vacances ?

Je constate très rapidement qu’il est difficile de lire du texte sur l’écran en 144Hz.

R.I.P l’anti-aliasing

Une recherche sur l’internet m’informe que le problème est largement constaté depuis la version Mojave avec des écrans externes non-Retina.

La solution consiste à désactiver l’option CGFontRenderingFontSmoothingDisabled. Voici la commande à saisir dans un Terminal :

defaults write -g CGFontRenderingFontSmoothingDisabled -bool FALSE

Il est nécessaire de se déconnecter / reconnecter (ou redémarrer) pour appliquer le changement. Et pour annuler le changement c’est facile, il suffit de changer la valeur FALSE à TRUE.
